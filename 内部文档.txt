1. 协议概述

本协议定义了通过标准音频输出设备向AI核心传输管理指令的基础技术标准。

1.1 设计原则

   a) 极简核心 - 只做编码和解码，不包含校验、握手、加密等功能
   b) 社区扩展 - 高级功能通过 mk/ 目录下的模块实现
   c) 向后兼容 - 核心协议定稿后不再修改

1.2 性能指标

┌─────────────────────────────────────────────────────────────────────────────┐
│ 参数名称              │ 规格值                                              │
├───────────────────────┼─────────────────────────────────────────────────────┤
│ 调制方式              │ 8FSK（8频移键控）                                   │
│ 符号数                │ 8                                                   │
│ 每符号比特            │ 3 bit                                               │
│ 符号时长              │ 80 毫秒                                             │
│ 有效数据传输率        │ 约 3.6 字节/秒                                      │
│ 工作频率范围          │ 1000 Hz - 3400 Hz                                   │
│ 采样率                │ 44100 Hz                                            │
│ 量化位深              │ 16 bit                                              │
│ 声道数                │ 1（单声道）                                         │
└───────────────────────┴─────────────────────────────────────────────────────┘

3. 数据帧结构

┌─────────────────────────────────────────────────────────────────────────────┐
│ 同步前导  │ 帧起始  │ 静音  │ 数据（UTF-8编码）                            │
│  500毫秒  │ 100毫秒 │ 40毫秒│                                               │
└─────────────────────────────────────────────────────────────────────────────┘

数据字段直接传输 UTF-8 编码的指令字符串，不包含头部、长度字段和校验码。

4. 字节编码规则

每个字节拆分为3个符号，每字节后跟40毫秒静音：

┌─────────────────────────────────────────────────────────────────────────────┐
│ 原始字节（8位）                                                             │
│ ┌───────┬───────┬───────┐                                                   │
│ │ 位7-5 │ 位4-2 │ 位1-0 │                                                   │
│ │ (3位) │ (3位) │ (2位) │                                                   │
│ └───────┴───────┴───────┘                                                   │
│      ↓        ↓        ↓                                                    │
│   符号1    符号2    符号3                                                    │
└─────────────────────────────────────────────────────────────────────────────┘

5. 核心接口

5.1 编码器（encoder.py）

输入: 指令字符串（UTF-8编码）
输出: cmd.wav（16位单声道，44100Hz采样率）

5.2 解码器（decoder.py）

输入: cmd.wav
输出: 指令字符串（UTF-8编码）

6.5 字符串模式

直接发送 UTF-8 字符串即可，无需特殊前缀。

示例:
"PING;STATUS"
"MOVE 100;TURN 90;STOP"

7. 扩展机制
